## 🎯 시스템 개요

### SSTDMS의 탄생 배경

조선 설계 업무는 수많은 기술 문서, 도면, 계산서, 승인서류 등을 다루는 복잡한 프로세스입니다. 전통적인 파일 기반 관리 방식은 다음과 같은 한계점을 가지고 있었습니다:

- 문서 버전 관리의 어려움
- 협업 과정에서의 정보 손실
- 승인 프로세스의 비효율성
- 보안 관리의 취약점
- 프로젝트 진행 상황 추적의 어려움

이러한 문제점들을 해결하기 위해 김봉정 수석설계사는 조선 설계 업무에 특화된 통합 문서 관리 시스템인 SSTDMS를 개발하게 되었습니다.

### 시스템의 핵심 철학

SSTDMS는 다음과 같은 핵심 철학을 바탕으로 설계되었습니다:

**1. 사용자 중심 설계**: 실제 조선 설계 업무를 수행하는 엔지니어들의 워크플로우를 깊이 이해하고, 이를 시스템에 자연스럽게 반영했습니다.

**2. 보안과 효율성의 균형**: 기업의 핵심 자산인 설계 정보를 안전하게 보호하면서도, 업무 효율성을 저해하지 않는 적절한 균형점을 찾았습니다.

**3. 확장성과 유연성**: 미래의 기술 발전과 업무 변화에 대응할 수 있도록 모듈화된 구조와 확장 가능한 아키텍처를 채택했습니다.

**4. 직관적 사용성**: 복잡한 기능을 단순하고 직관적인 인터페이스로 제공하여, 사용자의 학습 부담을 최소화했습니다.

### 시스템 아키텍처

SSTDMS는 현대적인 웹 기술 스택을 기반으로 구축되었습니다:

**프론트엔드**: React.js와 Tailwind CSS를 활용한 반응형 사용자 인터페이스
**백엔드**: Flask 프레임워크 기반의 RESTful API 서버
**데이터베이스**: SQLite를 기본으로 하되, 확장성을 위한 다중 데이터베이스 지원
**보안**: 암호화된 사용자 관리 시스템과 세션 기반 인증
**파일 관리**: 로컬 스토리지와 클라우드 스토리지 하이브리드 지원

**서버 환경 요구사항**:

- Node.js: 16.0 이상 (권장: 20.x LTS)
- 메모리: 최소 4GB RAM (권장: 8GB 이상)
- 저장공간: 최소 50GB (프로젝트 규모에 따라 확장)
- 네트워크: 안정적인 인터넷 연결

**클라이언트 환경 요구사항**:
- 웹 브라우저: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- 화면 해상도: 최소 1280x720 (권장: 1920x1080 이상)
- 네트워크: 안정적인 인터넷 연결 (최소 10Mbps)

### 설치 프로세스

SSTDMS의 설치는 백엔드 서버 구성과 프론트엔드 배포의 두 단계로 진행됩니다. 김봉정 수석설계사가 설계한 설치 프로세스는 기술적 복잡성을 최소화하면서도 보안성과 안정성을 보장하도록 구성되어 있습니다.

**1단계: 백엔드 서버 설정**

백엔드 서버는 Flask 프레임워크를 기반으로 구축되어 있으며, 다음과 같은 순서로 설치를 진행합니다:

```bash
# 프로젝트 디렉토리 생성
mkdir sstdms_erp && cd sstdms_erp

# 백엔드 환경 구성
cd sstdms_backend
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# 또는 venv\Scripts\activate  # Windows

# 필수 패키지 설치
pip install flask flask-cors sqlalchemy bcrypt cryptography pandas openpyxl
```

**2단계: 데이터베이스 초기화**

SSTDMS는 SQLite를 기본 데이터베이스로 사용하며, 필요에 따라 PostgreSQL이나 MySQL로 확장할 수 있습니다:

```bash
cd src
python init_users.py  # 사용자 테이블 초기화
python create_sbv_project.py  # 샘플 프로젝트 생성
```

**3단계: 보안 설정**

김봉정 수석설계사가 특별히 강조한 보안 기능을 활성화합니다:

```bash
# 보안 강화 사용자 관리 시스템 초기화
python -c "
from utils.user_manager import SecureUserManager
user_manager = SecureUserManager()
user_manager.initialize_default_users()
"
```

이 과정에서 암호화 키가 자동으로 생성되며, 사용자 비밀번호는 bcrypt 해시로 안전하게 저장됩니다.

**4단계: 프론트엔드 배포**

React 기반의 프론트엔드는 다음과 같이 구성됩니다:

```bash
cd ../sstdms_frontend
npm install
npm run build
```

**5단계: 서버 시작**

모든 구성이 완료되면 다음 명령으로 서버를 시작할 수 있습니다:

```bash
cd ../sstdms_backend/src
python main.py
```

서버는 기본적으로 5000번 포트에서 실행되며, 브라우저에서 `http://localhost:5000`으로 접속할 수 있습니다.

## 🔐 보안 시스템 상세 분석

### 다층 보안 아키텍처

김봉정 수석설계사가 설계한 SSTDMS의 보안 시스템은 다층 방어 전략을 채택하고 있습니다. 이는 조선 설계 정보라는 기업의 핵심 자산을 보호하기 위한 포괄적인 접근 방식입니다.

**1층: 네트워크 보안**
시스템의 첫 번째 방어선은 네트워크 레벨에서 구현됩니다. 방화벽 규칙을 통해 불필요한 포트를 차단하고, VPN을 통한 안전한 접근 경로를 제공합니다. 또한 DDoS 공격 방지를 위한 트래픽 모니터링과 제한 기능이 포함되어 있습니다.

**2층: 애플리케이션 보안**
Flask 프레임워크의 보안 기능을 최대한 활용하여 SQL 인젝션, XSS(Cross-Site Scripting), CSRF(Cross-Site Request Forgery) 등의 웹 애플리케이션 공격을 방어합니다. 모든 사용자 입력은 검증과 정화 과정을 거치며, 출력 시에도 적절한 인코딩이 적용됩니다.

**3층: 인증 및 권한 관리**
bcrypt 해시 알고리즘을 사용한 강력한 비밀번호 보호와 세션 기반 인증 시스템을 구현했습니다. 각 사용자의 권한은 역할 기반 접근 제어(RBAC) 모델에 따라 세밀하게 관리됩니다.

**4층: 데이터 암호화**
민감한 데이터는 Fernet 대칭 암호화를 사용하여 저장 시점과 전송 시점 모두에서 보호됩니다. 암호화 키는 별도의 보안 저장소에 관리되며, 정기적인 키 로테이션이 수행됩니다.

## 📊 프로젝트 관리 시스템

### 프로젝트 생명주기 관리

SSTDMS의 프로젝트 관리 시스템은 조선 설계 프로젝트의 특성을 깊이 이해한 김봉정 수석설계사의 실무 경험이 반영된 결과물입니다. 조선 프로젝트는 일반적으로 기본설계, 상세설계, 승인도서 작성의 단계를 거치며, 각 단계마다 수많은 문서와 도면이 생성됩니다.

**프로젝트 초기화 단계**:
새로운 프로젝트가 생성되면 시스템은 자동으로 표준화된 폴더 구조를 생성합니다. 이 구조는 Seastar Design의 오랜 경험과 국제 조선 표준을 바탕으로 설계되었습니다:

```
프로젝트명/
├── 01_기본설계/
│   ├── 일반배치도/
│   ├── 선형도/
│   ├── 구조도/
│   └── 의장도/
├── 02_상세설계/
│   ├── 선체구조/
│   ├── 의장설계/
│   ├── 기관설계/
│   └── 전기설계/
├── 03_승인도서/
│   ├── 선급승인도/
│   ├── 관청승인도/
│   └── 선주승인도/
└── 04_계산서/
    ├── 구조계산/
    ├── 복원성계산/
    └── 안전성평가/
```

**프로젝트 메타데이터 관리**:
각 프로젝트는 다음과 같은 상세 정보를 포함합니다:
- 프로젝트 코드 (예: SBV-2025-001)
- 선박 유형 (컨테이너선, 벌크선, 탱커, LNG선 등)
- 주요 제원 (길이, 폭, 깊이, 재화중량톤수)
- 발주처 정보 및 연락처
- 프로젝트 매니저 및 주요 담당자
- 계약 일정 및 마일스톤
- 적용 규정 및 선급 규칙

### 협업 및 워크플로우 관리

조선 설계는 본질적으로 다학제적 협업이 필요한 분야입니다. 선체 구조, 의장, 기관, 전기 등 각 분야의 전문가들이 긴밀하게 협력해야 하며, SSTDMS는 이러한 협업을 효과적으로 지원합니다.

**역할 기반 접근 제어**:
프로젝트 내에서도 세분화된 권한 관리가 가능합니다:
- 프로젝트 매니저: 전체 프로젝트 관리 및 승인 권한
- 분야별 리더: 해당 분야 문서의 생성 및 수정 권한
- 설계자: 담당 영역 문서의 생성 및 수정 권한
- 검토자: 문서 검토 및 코멘트 권한
- 조회자: 문서 조회 및 다운로드 권한

**승인 워크플로우**:
조선 설계에서는 엄격한 승인 프로세스가 필요합니다. SSTDMS는 다단계 승인 워크플로우를 지원합니다:
1. 설계자 작성 → 2. 분야 리더 검토 → 3. 프로젝트 매니저 승인 → 4. 고객 승인

각 단계에서 승인자는 상세한 검토 의견을 남길 수 있으며, 필요시 이전 단계로 반려할 수 있습니다.

### 일정 관리 및 간트차트

**자동 간트차트 생성**:
SSTDMS는 프로젝트의 마일스톤과 작업 일정을 바탕으로 자동으로 간트차트를 생성합니다. 이 기능은 김봉정 수석설계사가 수많은 프로젝트를 관리하면서 축적한 노하우가 반영된 것입니다.

간트차트는 다음과 같은 정보를 시각적으로 표현합니다:
- 각 설계 단계별 예상 소요 기간
- 작업 간의 의존성 관계
- 크리티컬 패스 (프로젝트 완료에 가장 중요한 경로)
- 현재 진행 상황 및 지연 위험도
- 리소스 할당 현황

**실시간 진행 상황 추적**:
프로젝트 팀원들이 작업을 완료하거나 문서를 업로드할 때마다 간트차트는 실시간으로 업데이트됩니다. 이를 통해 프로젝트 매니저는 언제든지 정확한 진행 상황을 파악할 수 있습니다.

**지연 예측 및 알림**:
시스템은 과거 프로젝트 데이터와 현재 진행 상황을 분석하여 잠재적인 지연을 예측합니다. 지연 위험이 감지되면 관련 담당자들에게 자동으로 알림이 발송됩니다.

## 📄 문서 및 도면 관리 시스템

### 통합 문서 관리 아키텍처

SSTDMS의 문서 관리 시스템은 조선 설계 업무의 복잡성과 다양성을 고려하여 설계되었습니다. 김봉정 수석설계사는 수년간의 실무 경험을 통해 조선 설계에서 다루어지는 문서의 특성을 깊이 이해하고, 이를 시스템에 반영했습니다.

**문서 분류 체계**:
조선 설계 문서는 그 성격과 용도에 따라 다음과 같이 분류됩니다:

1. **도면류 (Drawings)**:
   - 일반배치도 (General Arrangement)
   - 선형도 (Lines Plan)
   - 구조도면 (Structural Drawings)
   - 의장도면 (Outfitting Drawings)
   - 기관실 배치도 (Engine Room Arrangement)
   - 전기 계통도 (Electrical Diagrams)

2. **계산서류 (Calculations)**:
   - 구조 계산서 (Structural Calculations)
   - 복원성 계산서 (Stability Calculations)
   - 진동 해석서 (Vibration Analysis)
   - 유체역학 해석서 (CFD Analysis)

3. **승인서류 (Approval Documents)**:
   - 선급 승인도 (Class Approval Drawings)
   - 관청 승인도 (Flag State Approval)
   - 선주 승인도 (Owner's Approval)

4. **기술문서 (Technical Documents)**:
   - 설계 기준서 (Design Criteria)
   - 재료 명세서 (Material Specifications)
   - 용접 절차서 (Welding Procedures)
   - 검사 계획서 (Inspection Plans)

### 파일 버전 관리 시스템

조선 설계에서는 도면과 문서의 버전 관리가 매우 중요합니다. 설계 변경이 빈번하게 발생하며, 각 버전마다 승인 상태와 배포 이력을 정확히 추적해야 합니다.

**리비전 관리 체계**:
SSTDMS는 국제적으로 통용되는 리비전 관리 체계를 따릅니다:
- Rev.0: 초기 발행 (For Approval)
- Rev.1, Rev.2, ...: 승인 후 수정 (For Construction)
- Rev.A, Rev.B, ...: 건조 중 수정 (As Built)

**변경 이력 추적**:
각 리비전마다 다음 정보가 자동으로 기록됩니다:
- 변경 일시 및 변경자
- 변경 사유 및 상세 내용
- 영향받는 관련 문서 목록
- 승인자 및 승인 일시
- 배포 대상 및 배포 일시

**동시 편집 방지**:
여러 사용자가 동시에 같은 문서를 수정하는 것을 방지하기 위해 체크아웃/체크인 시스템을 구현했습니다. 한 사용자가 문서를 편집 중일 때는 다른 사용자는 읽기 전용으로만 접근할 수 있습니다.

### 고급 검색 및 필터링

대규모 조선 프로젝트에서는 수천 개의 문서가 생성될 수 있습니다. 필요한 문서를 빠르게 찾기 위해 SSTDMS는 강력한 검색 및 필터링 기능을 제공합니다.

**다차원 검색 기능**:
- 텍스트 검색: 파일명, 도면번호, 설명 등에서 키워드 검색
- 메타데이터 검색: 작성자, 작성일, 승인자, 승인일 등으로 검색
- 분류별 검색: 문서 유형, 분야, 승인 상태 등으로 필터링
- 고급 검색: 여러 조건을 조합한 복합 검색

**스마트 태깅 시스템**:
업로드되는 문서는 자동으로 분석되어 적절한 태그가 부여됩니다:
- 문서 유형 자동 인식
- 도면번호 자동 추출
- 관련 시스템 및 구역 자동 분류
- 중요도 및 우선순위 자동 평가

### 파일 형식 지원 및 변환

**지원 파일 형식**:
SSTDMS는 조선 설계에서 일반적으로 사용되는 모든 파일 형식을 지원합니다:
- CAD 파일: DWG, DXF, STEP, IGES
- 문서 파일: PDF, DOC, DOCX, XLS, XLSX, PPT, PPTX
- 이미지 파일: JPG, PNG, TIFF, BMP
- 압축 파일: ZIP, RAR, 7Z

**자동 변환 기능**:
업로드된 파일은 필요에 따라 자동으로 다른 형식으로 변환됩니다:
- CAD 파일 → PDF 변환 (조회 및 인쇄용)
- 이미지 파일 → 썸네일 생성
- 문서 파일 → 텍스트 추출 (검색 인덱싱용)

## 📊 엑셀 연동 및 데이터 관리

### 엑셀 기반 데이터 교환

조선 설계 업무에서 엑셀은 여전히 중요한 도구입니다. 설계 계산, 재료 명세서, 중량 계산서 등 많은 작업이 엑셀로 수행되며, SSTDMS는 이러한 엑셀 기반 워크플로우를 완벽하게 지원합니다.

**표준 엑셀 템플릿**:
김봉정 수석설계사는 Seastar Design의 오랜 경험을 바탕으로 다음과 같은 표준 엑셀 템플릿을 개발했습니다:

1. **도면 리스트 템플릿**:
   - 도면번호, 도면명, 분류, 리비전, 작성자, 작성일, 승인자, 승인일
   - 상태 (진행중/검토중/승인완료), 비고
   - 관련 문서 링크, 파일 경로

2. **재료 명세서 템플릿**:
   - 부품번호, 부품명, 재질, 규격, 수량, 단위
   - 공급업체, 단가, 총액
   - 납기, 검사 요구사항

3. **중량 계산서 템플릿**:
   - 구역별 중량 분류
   - 재료별 중량 집계
   - 무게중심 계산
   - 경량화 목표 추적

**일괄 업로드 기능**:
대량의 데이터를 효율적으로 처리하기 위해 엑셀 파일을 통한 일괄 업로드 기능을 제공합니다:
- 엑셀 파일 업로드 → 데이터 검증 → 오류 확인 → 일괄 등록
- 중복 데이터 자동 감지 및 처리
- 필수 필드 누락 검사
- 데이터 형식 자동 변환

**실시간 동기화**:
SSTDMS에서 수정된 데이터는 연결된 엑셀 파일과 실시간으로 동기화됩니다. 이를 통해 기존의 엑셀 기반 워크플로우를 유지하면서도 중앙 집중식 데이터 관리의 이점을 얻을 수 있습니다.

### 데이터 검증 및 품질 관리

**자동 데이터 검증**:
업로드되는 엑셀 데이터는 다음과 같은 검증 과정을 거칩니다:
- 필수 필드 완성도 검사
- 데이터 형식 및 범위 검증
- 참조 무결성 검사 (외래키 관계)
- 비즈니스 규칙 준수 검증

**데이터 품질 모니터링**:
시스템은 지속적으로 데이터 품질을 모니터링하고 개선 방안을 제시합니다:
- 중복 데이터 감지 및 정리
- 불완전한 데이터 식별
- 일관성 없는 명명 규칙 발견
- 데이터 품질 점수 산정

### 보고서 생성 및 분석

**자동 보고서 생성**:
SSTDMS는 프로젝트 데이터를 바탕으로 다양한 보고서를 자동으로 생성합니다:
- 프로젝트 진행 현황 보고서
- 도면 발행 현황 보고서
- 승인 지연 분석 보고서
- 리소스 활용 현황 보고서

**대시보드 및 시각화**:
복잡한 프로젝트 데이터를 직관적으로 이해할 수 있도록 다양한 차트와 그래프를 제공합니다:
- 프로젝트 진행률 게이지
- 분야별 작업 분포 파이차트
- 시간별 작업량 추이 그래프
- 품질 지표 트렌드 분석

## 🎨 워터마크 및 보안 기능

### 지적재산권 보호 시스템

조선 설계 정보는 기업의 핵심 지적재산권입니다. 김봉정 수석설계사는 이러한 중요성을 깊이 인식하고, SSTDMS에 강력한 지적재산권 보호 기능을 구현했습니다.

**동적 워터마크 시스템**:
SSTDMS의 워터마크 시스템은 단순한 로고 삽입을 넘어서 동적이고 지능적인 보호 기능을 제공합니다:

1. **사용자별 맞춤 워터마크**:
   - 다운로드하는 사용자의 이름과 이메일
   - 다운로드 일시 및 IP 주소
   - 고유한 추적 ID (문서 유출 시 추적 가능)

2. **문서별 보안 등급**:
   - 공개 (Public): 워터마크 없음
   - 내부 (Internal): 기본 워터마크
   - 기밀 (Confidential): 강화된 워터마크
   - 극비 (Top Secret): 최고 수준 보안 워터마크

3. **Seastar Design 브랜딩**:
   - 회사 로고 및 브랜드 컬러 적용
   - "World Shipbuilding & Offshore Design Provider" 슬로건 포함
   - 연락처 정보 및 저작권 표시

**워터마크 적용 기술**:
- PDF 파일: 투명도 조절 가능한 오버레이 워터마크
- 이미지 파일: 픽셀 레벨 워터마크 삽입
- CAD 파일: 메타데이터 워터마크 (파일 정보에 삽입)
- 문서 파일: 헤더/푸터 워터마크

### 접근 제어 및 권한 관리

**세분화된 권한 체계**:
SSTDMS는 문서별, 프로젝트별로 세분화된 권한 관리를 지원합니다:

1. **문서 레벨 권한**:
   - 조회 (View): 문서 내용 확인 가능
   - 다운로드 (Download): 파일 다운로드 가능
   - 편집 (Edit): 문서 수정 가능
   - 삭제 (Delete): 문서 삭제 가능
   - 공유 (Share): 다른 사용자와 공유 가능

2. **시간 기반 권한**:
   - 임시 권한: 특정 기간 동안만 유효한 권한
   - 조건부 권한: 특정 조건 충족 시에만 활성화
   - 승인 기반 권한: 관리자 승인 후 활성화

### 관리자 (Administrator) 매뉴얼

관리자는 SSTDMS의 전체적인 운영과 관리를 담당하는 최고 권한 사용자입니다. 김봉정 수석설계사가 설계한 관리자 기능은 시스템의 안정성과 보안성을 보장하면서도 효율적인 관리를 가능하게 합니다.

**시스템 관리 기능**:

1. **사용자 관리**:
   관리자는 모든 사용자 계정을 생성, 수정, 삭제할 수 있습니다. 새로운 직원이 입사하거나 조직 변경이 있을 때 신속하게 계정을 관리할 수 있습니다.
   
   - 신규 사용자 생성: 이메일, 이름, 부서, 직급, 권한 레벨 설정
   - 기존 사용자 정보 수정: 부서 이동, 승진, 권한 변경 등
   - 계정 비활성화: 퇴사자 또는 휴직자 계정 관리
   - 비밀번호 초기화: 사용자가 비밀번호를 분실했을 때 임시 비밀번호 발급
   - 권한 그룹 관리: 부서별, 프로젝트별 권한 그룹 생성 및 관리

2. **시스템 설정**:
   SSTDMS의 전반적인 동작을 제어하는 시스템 설정을 관리합니다.
   
   - 보안 정책 설정: 비밀번호 정책, 세션 타임아웃, 로그인 시도 제한
   - 파일 업로드 제한: 최대 파일 크기, 허용 파일 형식, 바이러스 검사 설정
   - 백업 설정: 자동 백업 주기, 백업 저장 위치, 보존 기간
   - 알림 설정: 이메일 서버 설정, 알림 템플릿, 발송 조건
   - 로고 및 브랜딩: 회사 로고, 색상 테마, 워터마크 설정

3. **모니터링 및 감사**:
   시스템의 상태를 실시간으로 모니터링하고 보안 감사를 수행합니다.
   
   - 시스템 성능 모니터링: CPU, 메모리, 디스크 사용량 추적
   - 사용자 활동 모니터링: 로그인 이력, 파일 접근 이력, 다운로드 통계
   - 보안 이벤트 추적: 실패한 로그인 시도, 권한 없는 접근 시도
   - 감사 로그 분석: 정기적인 보안 감사 및 컴플라이언스 확인
   - 용량 관리: 저장 공간 사용량 모니터링 및 정리

**고급 관리 기능**:

1. **데이터베이스 관리**:
   - 데이터베이스 백업 및 복원
   - 인덱스 최적화 및 성능 튜닝
   - 데이터 무결성 검사
   - 아카이빙 및 데이터 생명주기 관리

2. **통합 관리**:
   - 외부 시스템 연동 설정
   - API 키 관리 및 접근 제어
   - 웹훅 설정 및 모니터링
   - 데이터 동기화 상태 확인

### 등록자 (Registrar) 매뉴얼

등록자는 프로젝트 생성과 문서 관리의 핵심 역할을 담당합니다. Seastar Design의 설계팀에서 실제로 프로젝트를 이끌어가는 팀장급 인력들이 이 역할을 수행하며, 김봉정 수석설계사의 실무 경험이 깊이 반영된 기능들을 사용할 수 있습니다.

**프로젝트 관리 기능**:

1. **프로젝트 생성 및 설정**:
   새로운 조선 프로젝트를 시작할 때 등록자는 프로젝트의 기본 틀을 구성합니다.
   
   - 프로젝트 기본 정보 입력: 프로젝트명, 선박 유형, 주요 제원
   - 발주처 정보 관리: 선주사 정보, 담당자 연락처, 계약 조건
   - 프로젝트 일정 설정: 주요 마일스톤, 납기일, 중간 검토일
   - 팀 구성 및 역할 배정: 프로젝트 참여자 선정, 역할 및 권한 부여
   - 폴더 구조 생성: 표준 폴더 구조 적용 또는 커스터마이징

2. **문서 및 도면 관리**:
   프로젝트 진행 과정에서 생성되는 모든 문서와 도면을 체계적으로 관리합니다.
   
   - 파일 업로드 및 분류: 드래그 앤 드롭으로 간편한 업로드, 자동 분류
   - 리비전 관리: 도면 수정 시 자동 버전 관리, 변경 이력 추적
   - 승인 워크플로우 설정: 검토자 지정, 승인 단계 설정, 자동 알림
   - 배포 관리: 승인된 문서의 관련자 배포, 수신 확인 추적
   - 문서 상태 관리: 작성중/검토중/승인완료 등 상태 추적

3. **엑셀 연동 및 데이터 관리**:
   조선 설계에서 필수적인 엑셀 기반 작업을 시스템과 연동합니다.
   
   - 도면 리스트 관리: 엑셀 템플릿을 통한 도면 목록 일괄 등록
   - 재료 명세서 연동: 설계 변경 시 재료 목록 자동 업데이트
   - 중량 계산서 관리: 구역별 중량 데이터 통합 관리
   - 진도 관리: 작업 진행률 추적 및 보고서 생성
   - 품질 관리: 검사 체크리스트 및 품질 지표 관리

**고급 등록자 기능**:

1. **간트차트 및 일정 관리**:
   - 자동 간트차트 생성: 프로젝트 일정을 시각적으로 표현
   - 의존성 관리: 작업 간의 선후 관계 설정 및 관리
   - 리소스 할당: 인력 및 장비 배정, 과부하 상황 모니터링
   - 진행률 추적: 실시간 진행 상황 업데이트 및 지연 예측
   - 보고서 생성: 주간/월간 진행 보고서 자동 생성

2. **워터마크 및 보안 관리**:
   - 문서별 보안 등급 설정: 공개/내부/기밀/극비 등급 분류
   - 워터마크 커스터마이징: 프로젝트별 워터마크 설정
   - 접근 권한 관리: 사용자별, 문서별 세부 권한 설정
   - 다운로드 추적: 문서 다운로드 이력 모니터링
   - 보안 알림: 의심스러운 활동 감지 시 자동 알림

### 일반 사용자 (User) 매뉴얼

일반 사용자는 주로 설계 업무를 수행하는 엔지니어들로, 필요한 문서를 조회하고 다운로드하여 업무에 활용합니다. 김봉정 수석설계사는 실제 설계자의 관점에서 직관적이고 효율적인 사용자 인터페이스를 설계했습니다.

**기본 사용 기능**:

1. **프로젝트 조회 및 탐색**:
   할당받은 프로젝트에 접근하여 필요한 정보를 확인합니다.
   
   - 대시보드 활용: 개인화된 대시보드에서 담당 프로젝트 현황 확인
   - 프로젝트 상세 정보: 프로젝트 개요, 일정, 주요 연락처 조회
   - 진행 상황 추적: 전체 프로젝트 진행률 및 개인 작업 현황
   - 알림 확인: 새로운 문서, 승인 요청, 일정 변경 등 알림 수신
   - 즐겨찾기: 자주 사용하는 프로젝트 및 문서 즐겨찾기 등록

2. **문서 검색 및 조회**:
   방대한 프로젝트 문서 중에서 필요한 자료를 빠르게 찾습니다.
   
   - 통합 검색: 파일명, 내용, 메타데이터를 포함한 전체 검색
   - 고급 필터링: 문서 유형, 작성일, 작성자, 승인 상태 등으로 필터링
   - 폴더 탐색: 체계적인 폴더 구조를 통한 직관적 탐색
   - 최근 문서: 최근 조회한 문서 목록으로 빠른 재접근
   - 관련 문서: 현재 보고 있는 문서와 관련된 문서 자동 추천

3. **파일 다운로드 및 활용**:
   필요한 문서를 안전하게 다운로드하여 업무에 활용합니다.
   
   - 단일 파일 다운로드: 개별 파일의 최신 버전 다운로드
   - 다중 파일 다운로드: 여러 파일을 선택하여 ZIP 파일로 일괄 다운로드
   - 버전 선택: 필요에 따라 이전 버전의 문서 다운로드
   - 미리보기: 다운로드 전 문서 내용 미리보기
   - 다운로드 이력: 개인 다운로드 이력 확인 및 재다운로드

## 🔍 문제 해결 및 FAQ

### 일반적인 문제 해결

**로그인 관련 문제**:

Q: 로그인이 되지 않습니다.
A: 다음 사항을 확인해주세요:
1. 이메일 주소가 정확한지 확인 (대소문자 구분)
2. 비밀번호 입력 시 Caps Lock 상태 확인
3. 계정이 잠겨있는지 확인 (5회 실패 시 30분 잠금)
4. 네트워크 연결 상태 확인
5. 브라우저 쿠키 및 캐시 삭제 후 재시도

Q: 비밀번호를 잊어버렸습니다.
A: 관리자(admin@seastargo.com)에게 연락하여 비밀번호 초기화를 요청하세요. 보안을 위해 본인 확인 절차를 거친 후 임시 비밀번호가 발급됩니다.

**파일 업로드/다운로드 문제**:

Q: 파일 업로드가 실패합니다.
A: 다음 사항을 확인해주세요:
1. 파일 크기가 100MB를 초과하지 않는지 확인
2. 지원되는 파일 형식인지 확인
3. 네트워크 연결이 안정적인지 확인
4. 브라우저의 팝업 차단 설정 확인
5. 충분한 저장 공간이 있는지 확인

Q: 다운로드한 파일이 열리지 않습니다.
A: 다음 사항을 확인해주세요:
1. 해당 파일 형식을 지원하는 프로그램이 설치되어 있는지 확인
2. 파일이 완전히 다운로드되었는지 확인 (파일 크기 비교)
3. 바이러스 백신 프로그램이 파일을 차단하지 않았는지 확인
4. 다른 브라우저에서 다시 다운로드 시도

**성능 관련 문제**:

Q: 시스템이 느리게 동작합니다.
A: 다음 사항을 확인해주세요:
1. 인터넷 연결 속도 확인
2. 브라우저 캐시 및 쿠키 삭제
3. 다른 탭이나 프로그램을 많이 실행하고 있지 않은지 확인
4. 브라우저를 최신 버전으로 업데이트
5. 컴퓨터 재시작 후 재시도

### 고급 문제 해결

**권한 관련 문제**:

Q: "권한이 없습니다" 메시지가 나타납니다.
A: 이는 정상적인 보안 기능입니다. 다음과 같이 대응하세요:
1. 프로젝트 매니저나 등록자에게 권한 요청

3. 업무상 필요성을 명확히 설명하여 권한 신청
4. 임시 권한이 필요한 경우 승인 절차 진행

**데이터 동기화 문제**:

Q: 엑셀 파일과 시스템 데이터가 일치하지 않습니다.
A: 다음 절차를 따라주세요:

2. 로컬에서 수정한 내용을 새 파일에 반영

4. 데이터 검증 과정을 거쳐 오류 확인
5. 필요시 관리자에게 데이터 동기화 요청

**보안 관련 문제**:

2. 긴급한 경우 관리자에게 연락하여 수동 해제 요청
3. 향후 정확한 비밀번호 입력으로 예방
4. 비밀번호 변경을 통한 보안 강화 고려
