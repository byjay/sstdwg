# SSTDMS 현재 기능 및 아키텍처 분석 요약

## 1. 시스템 개요 및 목적
SSTDMS (Seastar Design Technical Document Management System)는 조선 및 해양 플랜트 설계 분야의 기술 문서, 도면, 계산서, 승인서류 등을 효율적으로 관리하기 위해 개발된 통합 문서 관리 시스템입니다. 전통적인 파일 기반 관리 방식의 한계점(버전 관리 어려움, 협업 비효율성, 보안 취약점 등)을 해결하고, 조선 설계 업무의 효율성과 정확성을 극대화하는 것을 목표로 합니다.

## 2. 핵심 철학
*   **사용자 중심 설계**: 실제 엔지니어의 워크플로우 반영
*   **보안과 효율성의 균형**: 설계 정보 보호와 업무 효율성 동시 추구
*   **확장성과 유연성**: 모듈화된 구조와 확장 가능한 아키텍처
*   **직관적 사용성**: 단순하고 직관적인 인터페이스 제공

## 3. 시스템 아키텍처
SSTDMS는 현대적인 웹 기술 스택을 기반으로 구축된 3-Tier 아키텍처를 따릅니다.
*   **프론트엔드**: React.js, Tailwind CSS (반응형 UI)
*   **백엔드**: Flask 프레임워크 기반 RESTful API 서버
*   **데이터베이스**: SQLite (기본), PostgreSQL/MySQL 확장 가능
*   **보안**: 암호화된 사용자 관리, 세션 기반 인증
*   **파일 관리**: 로컬 스토리지 및 클라우드 스토리지 하이브리드 지원

## 4. 주요 기능

### 4.1. 사용자 관리 및 권한 체계
*   **사용자 카테고리**: 관리자 (Administrator), 등록자 (Registrar), 일반 사용자 (User)로 구분.
*   **권한 체계**: 역할 기반 접근 제어 (RBAC) 모델을 통해 세분화된 권한 관리.
    *   **관리자**: 시스템 전체 관리, 사용자 계정/보안 정책/백업 관리, 감사 로그 모니터링.
    *   **등록자**: 프로젝트 생성/편집, 문서/도면 업로드/관리, 엑셀 파일 관리, 간트차트 관리, 워터마크 설정.
    *   **일반 사용자**: 할당된 프로젝트 조회, 문서/도면 조회/다운로드, 간트차트 조회, 개인 프로필 관리.

### 4.2. 보안 시스템
*   **다층 보안 아키텍처**: 네트워크, 애플리케이션, 인증/권한 관리, 데이터 암호화 (Fernet 대칭 암호화)의 다단계 방어.
*   **사용자 인증**: bcrypt 해시 비밀번호, 세션 기반 인증, 비밀번호 정책 (최소 8자, 복잡성, 재사용 방지), 계정 잠금 (5회 실패 시 30분), 세션 타임아웃 (60분), 동시 세션 제한 (3개).
*   **감사 로그**: 모든 사용자 활동 기록 (로그인/아웃, 파일 업로드/다운로드, 프로젝트/권한 변경 등), 365일 보존, 관리자만 접근 가능.
*   **데이터 보호 및 백업**: 파일 암호화 저장, 실시간 권한 검증, 다운로드 시 워터마크 자동 적용, 실시간/일일/주간 백업 전략.

### 4.3. 프로젝트 관리 시스템
*   **프로젝트 생명주기**: 표준화된 폴더 구조 자동 생성 (기본설계, 상세설계, 승인도서, 계산서 등).
*   **메타데이터 관리**: 프로젝트 코드, 선박 유형, 제원, 발주처, 담당자, 일정, 규정 등 상세 정보 관리.
*   **협업 및 워크플로우**: 프로젝트 내 세분화된 역할 기반 접근 제어, 다단계 승인 워크플로우 (설계자 → 분야 리더 → 프로젝트 매니저 → 고객).
*   **일정 관리 및 간트차트**: 마일스톤 기반 자동 간트차트 생성, 실시간 진행 상황 추적, 지연 예측 및 알림.

### 4.4. 문서 및 도면 관리 시스템
*   **통합 문서 관리**: 도면류, 계산서류, 승인서류, 기술문서 등 조선 설계 문서 분류 체계 지원.
*   **파일 버전 관리**: 리비전 관리 (Rev.0, Rev.1, Rev.A 등), 변경 이력 추적 (변경 일시, 변경자, 사유, 승인자 등), 동시 편집 방지 (체크아웃/체크인).
*   **고급 검색 및 필터링**: 텍스트, 메타데이터, 분류별, 고급 복합 검색, 스마트 태깅 시스템 (문서 유형, 도면번호, 시스템/구역 자동 인식).
*   **파일 형식 지원**: CAD (DWG, DXF, STEP, IGES), 문서 (PDF, DOC, XLS, PPT), 이미지 (JPG, PNG, TIFF), 압축 파일 (ZIP, RAR, 7Z) 등 광범위한 형식 지원.
*   **자동 변환 기능**: CAD → PDF, 이미지 → 썸네일, 문서 → 텍스트 추출.

### 4.5. 엑셀 연동 및 데이터 관리
*   **엑셀 기반 데이터 교환**: 표준 엑셀 템플릿 (도면 리스트, 재료 명세서, 중량 계산서), 엑셀 파일을 통한 일괄 업로드, 실시간 동기화.
*   **데이터 검증 및 품질 관리**: 필수 필드, 형식, 범위, 참조 무결성, 비즈니스 규칙 검증, 중복/불완전 데이터 감지.
*   **보고서 생성 및 분석**: 프로젝트 진행 현황, 도면 발행 현황, 승인 지연 분석 등 자동 보고서 생성, 대시보드 및 시각화 (차트, 그래프).

### 4.6. 워터마크 및 보안 기능
*   **지적재산권 보호**: 동적 워터마크 시스템 (사용자별 맞춤, 문서별 보안 등급, Seastar Design 브랜딩), 워터마크 적용 기술 (PDF, 이미지, CAD, 문서 파일).
*   **접근 제어**: 문서 레벨 권한 (조회, 다운로드, 편집, 삭제, 공유), 시간 기반 권한, 위치 기반 권한 (IP, 지역, VPN).
*   **감사 추적 및 모니터링**: 포괄적인 활동 로깅, 실시간 보안 모니터링 (대량 다운로드, 권한 없는 접근), 자동 알림 시스템 (이메일, SMS, 시스템 알림).

### 4.7. 확장성 및 통합 기능
*   **모듈화된 아키텍처**: 인증, 문서 관리, 프로젝트 관리, 보고서, 통합 모듈로 구성.
*   **외부 시스템 연동**: CAD (AutoCAD, AVEVA Marine 등), ERP (SAP, Oracle ERP 등), PLM (Siemens Teamcenter 등) 시스템 연동 지원.
*   **API 및 개발자 도구**: RESTful API, SDK (Python, JavaScript, .NET, Java), 웹훅 (실시간 이벤트 알림).
*   **클라우드 및 하이브리드 배포**: 클라우드 네이티브 설계 (Docker, Kubernetes, 마이크로서비스), 하이브리드 클라우드 지원, 멀티 클라우드 전략 (AWS, Azure, Google Cloud).

## 5. 결론
SSTDMS는 조선 설계 업무에 특화된 강력한 문서 및 프로젝트 관리 시스템으로, 보안, 협업, 데이터 관리, 확장성 등 다양한 측면에서 높은 완성도를 보여줍니다. 특히 김봉정 수석설계사의 실무 경험이 반영된 기능들은 실제 업무 환경에서 큰 효율을 가져올 것으로 예상됩니다. 모듈화된 아키텍처와 다양한 외부 시스템 연동 기능은 향후 시스템 확장의 기반이 됩니다.
